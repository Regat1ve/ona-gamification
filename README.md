# ona-gamification

Исследовательский прототип к выпускной квалификационной работе бакалавра
«Разработка подхода к повышению эффективности сотрудников малого бизнеса
на основе геймифицированного анализа взаимодействия в организации».

## Что делает прототип

Демонстрирует полный цикл авторского подхода на синтетических данных:

1. Генерирует логи коммуникаций модельной малой команды
   (25 сотрудников, 3 отдела, 4 рабочие недели).
2. Строит направленный взвешенный граф коммуникаций и считает базовые
   метрики организационного сетевого анализа (ONA): степенная центральность
   по входу и по выходу, центральность по посредничеству, коэффициент
   кластеризации.
3. Выполняет диагностику коммуникационных аномалий: изолированные узлы,
   перегруженные посредники, разрывы между отделами.
4. По матрице соответствия «аномалия - игровая механика» подбирает
   рекомендуемые геймификационные механики для каждой обнаруженной
   проблемы.
5. Симулирует два сценария вмешательства - адресный (по рекомендациям)
   и неадресный (массовая геймификация без опоры на диагностику).
6. Повторно строит граф и считает метрики, применяет парный
   t-критерий Стьюдента и U-критерий Манна-Уитни для проверки значимости
   изменений и сравнения сценариев.

## Структура репозитория

- `prototype.py` - единый исполняемый модуль со всеми этапами подхода.
- `prototype_summary.json` - автоматически формируемая сводка результатов
  запуска (пример для фиксированного зерна SEED = 42).
- `README.md` - этот файл.
- `.gitignore` - исключения для git.

## Запуск

Зависимости: Python 3.10+, numpy, networkx, pandas, scipy.

```
pip install numpy networkx pandas scipy
python prototype.py
```

Запуск выводит в консоль сводку по пяти этапам (генерация, диагностика,
рекомендации, сравнение сценариев, статистика) и сохраняет свежий
`prototype_summary.json` рядом с модулем.

## Воспроизводимость

Все случайные процессы инициализируются фиксированным зерном (SEED = 42
на старте модуля, seed_offset = 0 для первого замера и seed_offset = 100
для повторного). При одинаковой версии зависимостей результаты каждого
запуска совпадают вплоть до последней значащей цифры.

## Результаты вычислительного эксперимента

На стандартных параметрах запуска прототип демонстрирует статистически
значимое превосходство адресного подхода над неадресной базовой линией:

- средний прирост модельного KPI: +0,158 (адресный) против +0,052 (базовый);
- парный t-критерий (адресный, KPI): t = 6,90, p < 0,0001;
- U-критерий Манна-Уитни (адресный > базовый): U = 511, p = 0,00006;
- парный t-критерий (адресный, суммарная степенная центральность):
  t = 3,90, p = 0,00068.

Подробный разбор результатов и их интерпретация представлены в главе 3
квалификационной работы.
